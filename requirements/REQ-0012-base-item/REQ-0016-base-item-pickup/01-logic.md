## F-29: Подбор — условия срабатывания и логика

Подбор — переход `InWorld → InInventory`, зеркальный выбросу
([REQ-0015-base-item-drop](../REQ-0015-base-item-drop/README.md)).

| Условие | Правило |
|---------|---------|
| Триггер | **Автоматический**, без клавиши (грани «input» у фичи нет) |
| Близость | Игрок в пределах `PickupRange` от предмета (по умолчанию ≈ 1.1 м) |
| Прямая видимость | Между игроком и предметом нет преград — луч игрок→предмет не пересекает стену |
| Выбор при нескольких | Берётся **ближайший** подходящий предмет |
| Полный инвентарь | При 12/12 подбор невозможен; предмет остаётся в мире |
| «Взвод» (защита) | Предмет становится доступным для подбора только после того, как игрок хотя бы раз оказался **вне** его радиуса. Не даёт мгновенно поднять только что выброшенный предмет |

**Жизненный цикл подбора:**
1. Предмет лежит в мире (`InWorld`), уже «взведён» (игрок был вне радиуса).
2. Игрок входит в радиус с прямой видимостью, инвентарь не полон.
3. Предмет удаляется из мира и занимает свободный слот инвентаря (`InInventory`).
4. Иконка слота строится по 3D-модели предмета (та же модель, что была в мире).
5. Запускается анимация (F-30); по её завершении слот подсвечивается вспышкой.

**Границы:** ручной подбор по клавише, частичный подбор стопки, приоритет по типу — не входят (можно добавить позже).

## Конфигурируемые параметры

| Параметр | По умолчанию | Смысл |
|----------|--------------|-------|
| `PickupRange` | 1.1 м | Радиус срабатывания подбора |
| `PickupArmingRadius` | ≈ `PickupRange` + запас | Радиус «взвода»: выйдя за него, предмет становится доступным |
