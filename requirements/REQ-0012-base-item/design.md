## Предполагаемая структура

- **Конечный автомат состояний** (F-15): `InWorld`, `InInventory`, `Activated`. Естественная реализация — `enum ItemState` + методы-переходы `Pickup/Drop/Activate/Deactivate/Use`, проверяющие допустимость и управляющие бронированием слота (F-19a).
- **Данные (F-17):** класс `Item` (экземпляр) + ресурс/класс `ItemType` (неизменяемое описание типа: паттерн A/B, расходуемость, ссылки на визуальные ассеты). `ItemType` — кандидат в Godot `Resource` (`.tres`), чтобы типы править без перекомпиляции.
- **Бронирование слота (F-19a):** слот инвентаря несёт состояние `Empty / Occupied(item) / Reserved(item-in-hand)`. Активация → `Reserved`; уничтожение/выбрасывание → `Empty`; деактивация → `Occupied`.
- **3D-представление (InWorld):** `Node3D` со ссылкой на `.glb`; без физической коллизии со стенами (как у `Mob`), допускается `Area3D` для зоны подбора.
- **UI-представление:** иконка в слоте инвентаря + индикатор активированного предмета в HUD.

## Интеграция с существующим кодом

- Инвентарь (когда будет реализован) — владелец слотов; см. [REQ-0011-inventory/design.md](../REQ-0011-inventory/design.md).
- Мировая позиция клетки — через `MazeData` (world↔cell, [TECH_SPEC §7](../TECH_SPEC.md#7-coordinate-systems)).
- Управление мышью при активированном предмете паттерна B: ЛКМ — применение; учесть, что мышь захвачена (`Input.MouseMode = Captured`) — ЛКМ работает и в захваченном режиме.
