# REQ-0012-base-item — Реализация (Design / HOW)

> **Фича:** REQ-0012-base-item · **Тип документа:** реализация (HOW) · **Статус:** ✅ базово реализовано
> **Глобальный контекст:** [TECH_SPEC.md](../TECH_SPEC.md).

Базовая сущность реализована в минимальном, но полном по состояниям виде: три состояния
(InWorld / InInventory / Activated), оба паттерна использования (A/B) и бронирование слота.
Специфика конкретных предметов — в их требованиях ([REQ-0013](../REQ-0013-vintage-camera/),
[REQ-0017](../REQ-0017-photo/)).

---

## Структура кода

- **`src/Item.cs`** — экземпляр предмета (данные, F-17): `TypeId`, `DisplayName`, `Category`
  (`Consumable`/`Key`), `Usage` (`ImmediateA`/`ActivatedB`, F-18), `ModelPath`, `Icon`.
  - `Use()` — применение паттерна A (F-18): печатает лог, возвращает «уничтожить ли»
    (расходник). Реальные эффекты — у конкретных типов.
  - `virtual Node3D BuildModel()` — фабрика 3D-узла (для иконки и для предмета в мире);
    по умолчанию инстанцирует glb по `ModelPath`, подклассы переопределяют (см. `PhotoItem`).
- **`src/Inventory.cs`** — 12 слотов (данные), `TryAdd/PutAt/RemoveAt/Get/Count/IsFull`.
- **`src/InventoryHud.cs`** — хаб: состояния, переходы, ввод, отрисовка.
- **`src/WorldItem.cs`** — состояние InWorld (REQ-0014).

## Состояния и переходы (F-15, F-19, F-19a)

Состояние предмета не хранится полем — оно задаётся его «местом»:
- **InWorld** — предмет живёт как `WorldItem` в сцене (реестр `WorldItem.All`).
- **InInventory** — предмет в массиве слотов `Inventory`.
- **Activated** — предмет одновременно **в своём слоте** (слот занят → блокирует ячейку) и
  помечен полями `InventoryHud._activatedItem` / `_reservedSlot`. Так слот **забронирован**
  (F-19a): `TryAdd`/подбор его не займут, обычный drop по нему уходит в drop-activated.

Переходы в `InventoryHud`:

| Переход | Метод | Примечание |
|---------|-------|-----------|
| Активация (InInventory→Activated) | `ActivateSlot` | только `Usage=ActivatedB`; эксклюзивно (прежний деактивируется, F-19) |
| Деактивация (Activated→InInventory) | `Deactivate` | повторный выбор той же ячейки (toggle) |
| Выброс активированного (Activated→InWorld) | `DropActivated` | Shift по забронированной ячейке; слот освобождается |
| Уничтожение с заменой | `ConsumeActivated(replacement)` | камера→фото в тот же слот; фото→null (освободить) |
| Применение A (InInventory→…) | `ApplySlot`→`Item.Use` | расходник удаляется |
| Подбор (InWorld→InInventory) | `StartPickup` | [REQ-0016](./REQ-0016-base-item-pickup/) |
| Выброс (InInventory→InWorld) | `DropSlot`→`SpawnDrop` | [REQ-0015](./REQ-0015-base-item-drop/) |

При активации проигрывается жест взятия в руку `Player.PlayPickupGesture()` (клип `Interact`).

## Ввод (F-18)

В `InventoryHud._Input`: открытый инвентарь, выбор ряда (1–3) → ячейки (1–4). На ячейке:
- `Usage=ActivatedB` → `ActivateSlot` (или `Deactivate`, если это забронированная ячейка);
- иначе → `ApplySlot` (паттерн A).
- Shift на ячейке → выброс (`DropActivated` для забронированной, иначе `DropSlot`).
- `use_activated` (ЛКМ) — использование активированного паттерна B (диспетчеризуется по типу:
  фотоаппарат открывает видоискатель; фотография на ЛКМ не реагирует — она «входится» ходьбой).

## UI (F-16)

- Иконка слота — рендер `Item.BuildModel()` в `SubViewport` → `ViewportTexture`; камера посажена
  плотно (`FrameCamera`, FOV 30), модель почти заполняет ячейку (на 2K иконки были мелкими).
- Забронированный слот: приглушение + рамка `HandColor` + метка «рука».
- Индикатор активированного (компакт-панель): иконка предмета «в руке» с подсветкой
  (упрощённый UI-индикатор F-16; отдельный крупный HUD-индикатор у края экрана — отложен).

## Границы (НЕ реализовано)

- **`Item` — обычный C#-класс**, не `Resource`/`.tres` реестр типов (тип задаётся конструктором
  в коде). «Реестр типов» из наброска не заводился.
- Нет сериализации/сохранения между сессиями.
- UI-индикатор активированного — компактный (иконка в панели рюкзака), без отдельного крупного
  HUD-виджета у края экрана.
- Подсказки-имена при наведении/приближении — отложены.
