## F-26: Переход drop

Выброс — это переход предмета из владения игрока обратно в мир. Определён в
[REQ-0012-base-item/01-states-logic.md](../01-states-logic.md) как переход
`InInventory → InWorld` (и `Activated → InWorld` для активированного предмета).

| Аспект | Правило |
|--------|---------|
| Источник | Предмет в слоте инвентаря (`InInventory`) |
| Результат | Предмет удаляется из слота; слот **освобождается** |
| Появление в мире | Предмет переходит в `InWorld` (см. [REQ-0014](../REQ-0014-base-item-item-in-world/README.md)) в точке приземления |
| Точка приземления | Перед игроком, по направлению взгляда (`PlanarFacing`), на расстоянии `DropDistanceBodies × PlayerBodyDiameter` (по умолчанию 3 × 0.6 м ≈ 1.8 м) |
| Препятствие | Если по лучу вперёд стена ближе целевой дистанции — приземление сдвигается к стене (не доходя, с небольшим отступом) |
| Универсальность | Не зависит от типа/категории предмета: расходник и ключ выбрасываются одинаково |
| Активированный предмет | `Activated → InWorld` также снимает бронь слота (F-19a). Активация в руку не реализована → на практике выброс идёт из `InInventory` |

**Жизненный цикл выброса:**
1. Игрок выбирает ячейку и даёт команду выброса (F-27).
2. Если ячейка пуста — ничего не происходит.
3. Предмет удаляется из слота (слот свободен сразу).
4. Запускается анимация летящей звёздочки (F-28) от игрока к точке приземления.
5. По завершении анимации в точке приземления создаётся 3D-модель предмета (`InWorld`, [REQ-0014](../REQ-0014-base-item-item-in-world/README.md)).

**Границы:** подбор предмета обратно из мира (`InWorld → InInventory`) — отдельная задача (pickup, отложено).

## Конфигурируемые параметры

| Параметр | По умолчанию | Смысл |
|----------|--------------|-------|
| `DropDistanceBodies` | 3 | Дальность в «корпусах» игрока |
| `PlayerBodyDiameter` | 0.6 м | Эталон «корпуса» (диаметр игрока) |
