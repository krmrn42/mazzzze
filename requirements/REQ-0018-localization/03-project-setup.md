## F-37: Настройки проекта — секция internationalization

Для работы системы переводов проект Godot должен содержать соответствующие настройки. На момент постановки требования секция `[internationalization]` в `project.godot` отсутствует.

### Обязательные настройки

| Настройка (смысл) | Назначение |
|-------------------|-----------|
| Регистрация переводов | Скомпилированные ресурсы переводов (по одному на каждый язык) подключены к проекту, чтобы `TranslationServer` о них знал |
| Базовый язык (fallback) | Объявлен язык, на который приходит fallback-цепочка при отсутствии перевода (`en`) |
| Папка переводов | Переводные файлы хранятся в одном известном месте проекта (`res://translations/`) |

### Настройка для разработки

| Настройка (смысл) | Назначение |
|-------------------|-----------|
| Тестовая локаль | (Опционально) принудительный язык для проверки; в релизе пуста |

### Интеграция со сборкой

- Переводные ресурсы включаются в экспортируемый билд как обычные ресурсы Godot
- Добавление языка не требует изменения кода — только CSV + регистрация

### Связь с запуском

- Минимальный bootstrap при запуске устанавливает локаль из ОС (см. [F-36](./02-locale-logic.md)); настройки определяют, какие переводы доступны и кудаfallback приходит
