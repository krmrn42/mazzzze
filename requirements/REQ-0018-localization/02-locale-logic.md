## F-36: Выбор языка при запуске — локаль устройства + fallback

### Поведение по умолчанию

| Параметр | Значение |
|----------|----------|
| Источник языка при запуске | **Локаль устройства**: ПК / консоль / мобильный телефон, с которого запускается игра |
| Способ получения | Код локали ОС (например, `ru_RU`, `en_US`) |
| Когда применяется | При каждом запуске, до первого отображения UI |
| Сохранение между сессиями | Не требуется — каждый запуск заново берёт локаль устройства |

### Цепочка fallback

Если для локали устройства перевода нет или он неполон, язык определяется по цепочке:

| Шаг | Условие | Результат |
|-----|---------|-----------|
| 1 | Есть перевод для полной локали (напр. `en_US`) | Используется он |
| 2 | Есть перевод для языка без региона (`en`) | Используется он |
| 3 | Нет перевода для языка | **Базовый язык (`en`)** — гарантирует, что игрок не увидит сырой ключ |

### Гарантия отсутствия «сырых» ключей

Поскольку ключи абстрактные ([F-35](./01-translation-data.md)), показ ключа вместо строки считается ошибкой. Цепочка всегда оканчивается на базовом языке (`en`), перевод для которого полон по определению.

### Принудительный язык для тестирования

| Параметр | Значение |
|----------|----------|
| Назначение | Проверить любой язык во время разработки, не меняя локаль ОС |
| Способ | Настройка проекта «тестовая локаль» (`internationalization/locale/test`) — если задана, перекрывает локаль устройства |
| В релизе | Пуста (не задана) — чтобы всегда работала локаль устройства |

### Границы (что НЕ входит)

- **Ручное переключение языка игроком** (меню настроек, горячая клавиша) — не требуется; язык следует за устройством
- **Сохранение выбранного языка** между сессиями — не требуется (нечего сохранять)
- **Переключение языка «на лету»** без перезапуска — не требуется (язык фиксируется при запуске)
- **Перевод ассетов** (текстур, аудио) через `translation_remaps` — не входит; только строки
