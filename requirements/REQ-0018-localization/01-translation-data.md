## F-35: Переводные ресурсы — формат и хранение

Строковые ресурсы игры хранятся и переводятся стандартными средствами Godot.

### Принцип ключей

| Параметр | Значение |
|----------|----------|
| Способ адресации строк | **Абстрактные ключи** (идентификаторы), а не исходные строки |
| Вид ключа | `UPPER_SNAKE_CASE`, с префиксом домена/фичи (например, `INV_BACKPACK_COUNT`, `ITEM_NAME_VINTAGE_CAMERA`) |
| Строка в коде | Вызов перевода по ключу возвращает строку на текущем языке; сам ключ игроку никогда не показывается |
| Следствие | Перевод должен существовать для **каждого** ключа в **базовом** языке — иначе сработает fallback (см. [F-36](./02-locale-logic.md)), а показ сырого ключа считается ошибкой |

### Формат и расположение

| Параметр | Значение по умолчанию | Смысл |
|----------|----------------------|-------|
| Технология | Встроенная в Godot система переводов | CSV → скомпилированные ресурсы `Translation` |
| Папка переводов | `res://translations/` | Единое место для переводных файлов |
| Имя источника | `strings.csv` | Единый источник переводов |
| Структура CSV | Первая колонка — ключ; каждая следующая колонка — код языка (`ru`, `en`, …) | Импорт компилирует по одному `Translation` на язык |
| Регистрация | Скомпилированные ресурсы подключаются в настройках проекта | Чтобы `TranslationServer` о них знал (см. [F-37](./03-project-setup.md)) |

### Языки

| Параметр | Значение |
|----------|----------|
| Базовый (fallback) язык | **Английский (`en`)** — обязан содержать перевод для **каждого** ключа без исключений |
| Второй язык | **Русский (`ru`)** |
| Расширение | Добавление языка = новая колонка в CSV + регистрация; набор языков открытый |

### Полнота и целостность

- Каждый ключ обязан присутствовать со значением в колонке базового языка (`en`) обязательно; в остальных языках — желательно (иначеfallback на `en`)
- Удаление ключа из CSV должно сопровождаться удалением всех обращений к нему из кода
- Повторяющиеся ключи в CSV недопустимы

> Точный вид CSV-файла и примеры строк — в [design.md](./design.md).
