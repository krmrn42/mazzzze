## US-18: Локализация интерфейса по локали устройства

**Как** игрок,
**я хочу** чтобы интерфейс игры отображался на языке моего устройства (ПК / консоль / мобильный),
**чтобы** пользоваться игрой без языкового барьера.

**Критерии приёмки:**
- Все видимые пользователю строки UI проходят через систему переводов Godot (`TranslationServer.Translate` / `Node.Tr`) с использованием **абстрактных ключей** ([F-35](./01-translation-data.md), [F-38](./04-code-usage.md))
- Переводные ресурсы хранятся в стандартном для Godot формате (CSV → скомпилированные `Translation`), с колонками для каждого поддерживаемого языка ([F-35](./01-translation-data.md))
- При запуске игры активный язык **по умолчанию равен локали устройства** (`OS.GetLocale()`): запуск на русскоязычном устройстве → русский UI, на англоязычном → английский UI ([F-36](./02-locale-logic.md))
- Если для локали устройства нет перевода — срабатывает цепочка fallback до базового языка (английского), так чтобы пользователь **никогда не видел сырой ключ** ([F-36](./02-locale-logic.md))
- Настройки проекта (`project.godot`, секция `[internationalization]`) регистрируют переводы и объявляют базовый язык fallback ([F-37](./03-project-setup.md))
- Отладочные сообщения (`GD.Print`) **не** локализуются — они для разработчиков ([F-38](./04-code-usage.md))
- Существует способ принудительно задать язык для тестирования (без смены локали ОС) ([F-36](./02-locale-logic.md))

> Ручное переключение языка игроком (меню настроек) — **не входит** в это требование (см. границы в [02-locale-logic.md](./02-locale-logic.md) и [design.md](./design.md)).
