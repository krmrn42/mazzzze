# Maze Explorer — Текущие назначения клавиш (Keybindings)

> **Документ:** REQ-0004 · **Тип:** справочник ввода (WHAT + ссылка на реализацию)
> **Аудитория:** геймдизайнеры, продакт-оунеры, AI-агенты.
> **Снимок текущего состояния.** Обновлять при любом изменении карты ввода.

Единый список клавиш/действий, реально работающих в игре сейчас. Источники истины: секция `[input]` в `project.godot` (переназначаемые действия) и обработчики `_Input` в коде (зашитые комбинации).

---

## Перемещение (camera-relative)

Движение относительно направления взгляда камеры (в плоскости пола). Действия — в `project.godot`, обрабатываются в `src/Player.cs._PhysicsProcess` через `Input.GetVector`.

| Действие | Клавиша(и) | Геймпад | Действие (action) |
|----------|-----------|---------|-------------------|
| Вперёд | **W** / **↑** | кнопка 3 | `move_forward` |
| Назад | **S** / **↓** | кнопка 4 | `move_back` |
| Влево | **A** / **←** | кнопка 2 | `move_left` |
| Вправо | **D** / **→** | кнопка 1 | `move_right` |

Deadzone: 0.2. Прыжка в текущей версии нет (см. «Примечания»).

## Камера / обзор

Обрабатываются в `src/Player.cs._Input`. Мышь захвачена (`Input.MouseMode = Captured`), курсор скрыт.

| Действие | Ввод | Поведение |
|----------|------|-----------|
| Осмотр по горизонтали | Движение мыши ←/→ | Yaw камеры (без ограничений, 360°) |
| Осмотр по вертикали | Движение мыши ↑/↓ | Pitch камеры, ограничен [−85°, −25°] |
| Приблизить камеру | Колёсико **на себя** (WheelUp) | Дистанция → `MinZoom` (6.0) |
| Отдалить камеру | Колёсико **от себя** (WheelDown) | Дистанция → `MaxZoom` (14.0) |

> Колёсико зумит камеру только БЕЗ зажатого Ctrl. При зажатом Ctrl `Player` игнорирует wheel — его забирает мини-карта.

## Мини-карта

Действие `minimap_toggle` — в `project.godot`; зум — в `src/Minimap.cs._Input`.

| Действие | Ввод | Поведение |
|----------|------|-----------|
| Переключить ориентацию | **Tab** | camera-up ↔ north-up (`minimap_toggle`) |
| Приблизить миникарту | **Ctrl** + колёсико «на себя» | `_cellsRadius` → `MinCellsRadius` (5) |
| Отдалить миникарту | **Ctrl** + колёсико «от себя» | `_cellsRadius` → `MaxCellsRadius` (28) |

Событие Ctrl+wheel поглощается (`SetInputAsHandled`). См. [REQ-0010-minimap/03-input.md](./REQ-0010-minimap/03-input.md).

## Инвентарь

Действие `inventory_toggle` — в `project.godot`; цифровые клавиши — в `src/InventoryHud.cs._Input`.

| Действие | Ввод | Поведение |
|----------|------|-----------|
| Открыть/закрыть | **I** (одиночное нажатие) | toggle панели (`inventory_toggle`) |
| Выбрать ряд (при открытом) | **1** / **2** / **3** | подсветка ряда |
| Выбрать ячейку и применить/активировать | **1** / **2** / **3** / **4** (второе нажатие) | паттерн A — применение (`Item.Use`); паттерн B — взять «в руку» (активация, см. ниже) |

Цифры: верхний ряд (`Key1`…`Key4`) и цифровой блок (`Kp1`…`Kp4`). Клавиши читаются по **физическому** коду (устойчиво к раскладке и к зажатому Shift). Игра при открытии НЕ ставится на паузу. См. [REQ-0011-inventory/03-input.md](./REQ-0011-inventory/03-input.md).

## Выбрасывание предмета (drop)

Запускается из **открытого** инвентаря; обрабатывается в `src/InventoryHud.cs._Input`. Два равнозначных способа.

| Действие | Ввод | Поведение |
|----------|------|-----------|
| Выбросить ячейку (модификатор) | Выбрать ряд (**1**–**3**), затем **Shift + 1**…**4** | вместо применения — выброс ячейки |
| Выбросить под курсором | **G** | выброс предмета в текущей выбранной (подсвеченной) ячейке |

«Курсор» — последняя выбранная ячейка, остаётся подсвеченной для выброса по **G**. Если ячейка пуста — ничего не происходит. Предмет вылетает «звёздочкой» и приземляется перед игроком 3D-моделью. См. [REQ-0015-base-item-drop/02-input.md](./REQ-0012-base-item/REQ-0015-base-item-drop/02-input.md) и [REQ-0014-base-item-item-in-world](./REQ-0012-base-item/REQ-0014-base-item-item-in-world/README.md).

## Подбор предмета (pickup)

**Без клавиши — автоматический.** Подбор срабатывает сам, когда игрок подходит достаточно близко к лежащему предмету при прямой видимости (нет стены между ними) и инвентарь не полон. Обрабатывается в `src/InventoryHud.cs._PhysicsProcess`. См. [REQ-0016-base-item-pickup](./REQ-0012-base-item/REQ-0016-base-item-pickup/README.md).

## Активация и использование «в руке» (паттерн B)

Предметы паттерна B (фотоаппарат — [REQ-0013](./REQ-0013-vintage-camera/README.md), фотография — [REQ-0017](./REQ-0017-photo/README.md), теннисный мячик — [REQ-0021](./REQ-0021-tennis-ball/README.md)) берутся «в руку». Обрабатывается в `src/InventoryHud.cs`.

| Действие | Ввод | Поведение |
|----------|------|-----------|
| Активировать / деактивировать | В открытом инвентаре: ряд (**1**–**3**) → ячейка (**1**–**4**) с предметом паттерна B | взять в руку (слот бронируется, F-19a) / вернуть в слот (повторный выбор той же ячейки, toggle) |
| Выбросить активированный | Та же ячейка → **Shift + 1**…**4** | Activated → InWorld, слот освобождается |
| Использовать фотоаппарат | **ЛКМ** (`use_activated`, кнопка мыши 1) | открыть видоискатель, отсчёт 3→2→1, снимок → фотография |
| Зарядить бросок мячика | **Зажать ЛКМ** (`use_activated`, когда активирован мяч) | мяч в руке; растёт сила броска (до максимума); движение разрешено (медленнее) 🟡 [REQ-0021](./REQ-0021-tennis-ball/README.md) |
| Бросить мячик | **Отпустить ЛКМ** (после заряда) | выпуск снаряда по дуге (направление — yaw камеры); мяч → InWorld, слот освобождается; не расходуется 🟡 [REQ-0021](./REQ-0021-tennis-ball/README.md) |
| Минимальный бросок мячика | Короткий **тап ЛКМ** | бросок с минимальной силой 🟡 [REQ-0021](./REQ-0021-tennis-ball/README.md) |
| Отменить заряд мячика | **RMB** (кнопка мыши 2) во время заряда | заряд сбрасывается, мяч остаётся в руке, броска нет 🟡 [REQ-0021](./REQ-0021-tennis-ball/README.md) |
| Войти в фотографию | Удерживать **W/↑**, реально идя вперёд | накопление входа (≈2 с) → перенос в запечатлённую точку (F-33) |

Во время видоискателя мышь крутит **только yaw** (наклон pitch заблокирован); повторная ЛКМ игнорируется; открытие инвентаря заблокировано. ЛКМ — действие `use_activated` в `project.godot`. Вход в фотографию не имеет отдельной клавиши — это устойчивый жест «идти вперёд» (вариант паттерна B). 🟡 — запланировано, не реализовано.

## Примечания

- **`jump` (Space / геймпад-кнопка 0)** — действие существует в `project.godot`, но кодом **не используется**: прыжка нет, персонаж подчиняется только гравитации ([REQ-0003-core/07-constraints.md, F-08](./REQ-0003-core/07-constraints.md)). Переназначаемая «висящая» кнопка.
- **Мышь захвачена** всё время игры (`Input.MouseMode = Captured` в `src/Player.cs._Ready`), поэтому клики мышью по HUD не предусмотрены — ввод инвентаря цифровой.
- **`Input.UseAccumulatedInput = false`** (там же) — фикс обзора мышью при зажатых клавишах движения на Linux/X11 (не бинд, но важная настройка ввода).
- **Геймпад — частичный:** в action-карте есть только кнопки (цифровое движение), осей для обзора/зума нет. Обзор и зум — только мышь.
