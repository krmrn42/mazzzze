# REQ-0011-inventory — Реализация (Design / HOW)

> **Фича:** REQ-0011-inventory · **Тип документа:** реализация (HOW) · **Статус:** ✅ реализовано
> **Глобальный контекст:** [TECH_SPEC.md](../TECH_SPEC.md).

---

## Структура кода

- **`src/InventoryHud.cs`** (`Control`, узел `HUD/Inventory` в `main.tscn`): виджет в правом нижнем углу. Два состояния отрисовки через `_Draw`: compact (мешок + «N/12») и expanded (сетка 3×4). Стиль процедурный, в тон `Minimap` — без текстурных ассетов.
- **`src/Inventory.cs`** (`RefCounted`): чистая модель на 12 слотов (`Rows=3 × Cols=4`), `TryAdd`/`PutAt`/`RemoveAt`/`Get`, ограничение ёмкости (F-12).
- **`src/Item.cs`** (`RefCounted`): минимальная сущность предмета — `TypeId`, `DisplayName`, `Category` (`Consumable`/`Key`), `Icon`, `Use()`. Это подмножество REQ-0012 (только для инвентаря).
- **Иконка предмета:** для «Старинного фотоаппарата» модель `art/old_kodak_camera.glb` рендерится в `SubViewport` (`OwnWorld3D`, авто-кадрирование камеры по AABB) → `ViewportTexture` рисуется в ячейку. Так требование «разные иконки по типу» использует реальную 3D-модель. Иконка заполняет ячейку почти целиком — отступ от краёв всего **3% высоты ячейки**.

## Ввод (F-14)

- Действие `inventory_toggle` = **I** (в `[input]` `project.godot`). Двойное нажатие детектируется по интервалу (`DoublePressWindow`, окно 0.4 с) в `_Input`.
- Цифры 1–4 читаются напрямую по `Keycode` (верхний ряд + Kp) при открытом инвентаре: первая (1–3) выбирает ряд (подсветка), вторая (1–4) — колонку и применяет предмет (`ApplySlot` → `Item.Use`, вспышка через `_Process`).
- Игра не ставится на паузу, мышь остаётся захваченной.

## Решённые вопросы

- **Хранение состояния:** модель `Inventory` живёт в самом виджете (`InventoryHud`). Синглтон-узел не понадобился.
- **Анимация вспышки:** таймер в `_Process` (`_flashT`), без `AnimationPlayer`/`Tween`.

## Границы реализации (вне REQ-0011)

- **Применение = паттерн A** (немедленно из инвентаря). `Item.Use()` для расходника освобождает слот, для ключа — оставляет. Паттерн B (активация в руку по ЛКМ), бронь слота (F-19a), UI-индикатор активного предмета — часть REQ-0012, не реализованы.
- **Один засеянный предмет:** «Старинный фотоаппарат» кладётся в слот 0 при старте (`InventoryHud._Ready`). Реальная механика фотоаппарата (REQ-0013) не реализована — `Use()` для него заглушка.
- **Подбор (pickup) и 3D-предметы в мире** (REQ-0012) — не реализованы; предмет засевается напрямую в инвентарь.
- **Клик мышью по иконке в углу** (альтернатива F-14) — отложено: курсор захвачен во время игры.
