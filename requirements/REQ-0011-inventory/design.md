## Предполагаемая структура

- **HUD-виджет:** `Control`/`TextureRect` в существующей `HUD` (`CanvasLayer`, `main.tscn`), правый нижний угол. Состояния: compact (закрыт) / expanded (сетка 3×4).
- **Модель данных:** массив/слот-лист на 12 записей; каждый слот ссылается на экземпляр предмета (см. [REQ-0012-base-item/02-data-logic.md](../REQ-0012-base-item/02-data-logic.md)) либо пуст/забронирован.
- **Ввод:** действие `inventory_toggle` (двойное I) + цифровые клавиши 1–4; добавить в `[input]` секцию `project.godot`. Двойное нажатие I — детектировать по таймеру (как combo), либо по интервалу между `InputEventKey` для `Key.I`.
- **Интеграция с предметами:** фактическое применение/активацию делегировать сущности предмета (F-18, паттерны A/B). Инвентарь отвечает только за навигацию по слотам и подсветку.

## Открытые вопросы реализации

- Где хранить состояние инвентаря (узел-синглон, ресурс, модель в коде) — решить при реализации.
- Анимация подсветки/вспышки ячейки — `AnimationPlayer` или `Tween`.
- Взаимодействие с курсором мыши (захват мыши при открытом инвентаре?) — пока ввод цифровой, мышь остаётся захваченной.
