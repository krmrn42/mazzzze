# Requirements Catalog — Maze Explorer

Каталог требований игры. **WHAT** (что игра делает) живёт в файлах требований; **HOW** (как реализовано) — в [TECH_SPEC.md](./TECH_SPEC.md).

## Соглашения

- **Имя файла/папки фичи:** `REQ-NNNN-<slug>`, где `NNNN` — 4-значный ID. Для фич с одним User Story ID совпадает с номером US (`US-10 → REQ-0010`). Ядро и мета-документы используют зарезервированные младшие ID (0000–0009).
- **Папка фичи** содержит: `README.md` (обзор + User Story + критерии приёмки + статус + карта ID), пронумерованные facet-файлы по граням (`NN-logic/ui/visual/input/data/animation.md` — это WHAT) и `design.md` (это HOW, ссылается на TECH_SPEC).
- **Семантические ID `US-NN` и `F-NN` не перенумеровываются** — это опорные точки в коде (`src/*.cs`) и TECH_SPEC.

## Грани facet-файлов

| Грань | Что описывает |
|-------|---------------|
| `logic` | правила, конечные автоматы, алгоритмы (ЧТО делается) |
| `ui` | HUD, виджеты, 2D-представление |
| `visual` | 3D-look, материалы, модели |
| `input` | карта ввода, управление |
| `data` | структура данных, поля |
| `animation` | анимации (где применимо) |

## Связи между фичами

- **Инвентарь (REQ-0011)** ↔ **Базовый предмет (REQ-0012)**: рюкзак предоставляет слоты, предмет занимает слот (вкл. бронирование при активации, F-19a).
- **Фотоаппарат (REQ-0013)** наследуется от **Базового предмета (REQ-0012)** (паттерн B, одноразовый).
- **Фотография (REQ-0017)** наследуется от **Базового предмета (REQ-0012)** (паттерн B — вариант «вход движением вперёд», одноразовый).
- **Фотоаппарат (REQ-0013)** при успешном использовании создаёт **Фотографию (REQ-0017)** в освободившемся (забронированном) слоте (передача слота). Фотография хранит точку и направление съёмки и позволяет «войти» в них, переместив игрока.
- **Фотография (REQ-0017)** опирается на мир/камеру/ввод из **Ядра (REQ-0003)** и на стриминг чанков (`ChunkManager`); визуальный переход маскирует подгрузку чанков при дальнем переносе.
- **Интернационализация (REQ-0018)** — сквозная инфраструктура: все видимые строки UI всех фич проходят через систему переводов Godot по **абстрактным ключам**; язык по умолчанию = локаль устройства (`OS.GetLocale()`), fallback на `en`. Применяется к UI-строкам **Инвентаря (REQ-0011)** (есть строки «Рюкзак», «выбросить») и к будущим строкам **Фотоаппарата (REQ-0013)** и **Фотографии (REQ-0017)** (имена предметов, подсказки). Отладочные `GD.Print` локализацией не затрагиваются.
- **Выбрасывание (REQ-0015)** — переход `InInventory → InWorld` базового предмета (REQ-0012); запускается из **Инвентаря (REQ-0011)** и создаёт **Предмет в мире (REQ-0014)** в точке приземления.
- **Поднять (REQ-0016)** — обратный переход `InWorld → InInventory`; зеркало выброса (REQ-0015), автоматически забирает **Предмет в мире (REQ-0014)** в **Инвентарь (REQ-0011)**.
- **Предмет в мире (REQ-0014)** уточняет состояние `InWorld` из **REQ-0012** и служит домом для анимаций предмета в мире.
- **Мини-карта (REQ-0010)** и **Инвентарь (REQ-0011)** — раздельные HUD-виджеты в одной `HUD` (`main.tscn`).
- **Ядро (REQ-0003)** определяет мир/камеру/ввод, на которые опираются все остальные фичи.
- **Базовый монстр (REQ-0019)** — шаблон монстра (восприятие, конечный автомат, доставка урона, реестр). Опирается на **Ядро (REQ-0003)** (мир/`MazeData`/чанки). Зависимости — будущие фичи: система здоровья игрока, спавнер монстров, теннисный мячик ([IDEA-0025](../ideas/items/IDEA-0025-tennis-ball.md)).
- **Ifrit (REQ-0020)** наследуется от **Базового монстра (REQ-0019)** (первый конкретный тип): ифрит, контактный режим, патруль сегмента, преследование с обрывом по расстоянию (по умолчанию 1 сегмент = 1 чанк стриминга).
- **Теннисный мячик (REQ-0021)** наследуется от **Базового предмета (REQ-0012)** (паттерн B, **многоразовый** — не расходуется). Воздействует на **Базового монстра (REQ-0019)**: стан при прямом попадании (монстр видел бросок) / отвлечение (промах, но мяч замечен). Бросок — новый переход `Activated → InWorld` через физический снаряд (≠ мягкая дуга `DropProjectile` из REQ-0015). Зависит от REQ-0019 для эффектов на монстров.
- **Окружения-киты стен (REQ-0022)** опирается на **Ядро (REQ-0003)**: рендерится поверх той же сетки клеток лабиринта и того же стриминга чанков (`MazeData`/`ChunkManager`/`Chunk`), меняя только визуальное представление стеновых клеток (скальные меши вместо гладкой коробки) — геометрия лабиринта, коллизия и проходимость не затронуты.
