# Requirements Catalog — Maze Explorer

Каталог требований игры. **WHAT** (что игра делает) живёт в файлах требований; **HOW** (как реализовано) — в [TECH_SPEC.md](./TECH_SPEC.md).

## Соглашения

- **Имя файла/папки фичи:** `REQ-NNNN-<slug>`, где `NNNN` — 4-значный ID. Для фич с одним User Story ID совпадает с номером US (`US-10 → REQ-0010`). Ядро и мета-документы используют зарезервированные младшие ID (0000–0009).
- **Папка фичи** содержит: `README.md` (обзор + User Story + критерии приёмки + статус + карта ID), пронумерованные facet-файлы по граням (`NN-logic/ui/visual/input/data/animation.md` — это WHAT) и `design.md` (это HOW, ссылается на TECH_SPEC).
- **Семантические ID `US-NN` и `F-NN` не перенумеровываются** — это опорные точки в коде (`src/*.cs`) и TECH_SPEC.

## Грани facet-файлов

| Грань | Что описывает |
|-------|---------------|
| `logic` | правила, конечные автоматы, алгоритмы (ЧТО делается) |
| `ui` | HUD, виджеты, 2D-представление |
| `visual` | 3D-look, материалы, модели |
| `input` | карта ввода, управление |
| `data` | структура данных, поля |
| `animation` | анимации (где применимо) |

## Связи между фичами

- **Инвентарь (REQ-0011)** ↔ **Базовый предмет (REQ-0012)**: рюкзак предоставляет слоты, предмет занимает слот (вкл. бронирование при активации, F-19a).
- **Фотоаппарат (REQ-0013)** наследуется от **Базового предмета (REQ-0012)** (паттерн B, одноразовый).
- **Выбрасывание (REQ-0015)** — переход `InInventory → InWorld` базового предмета (REQ-0012); запускается из **Инвентаря (REQ-0011)** и создаёт **Предмет в мире (REQ-0014)** в точке приземления.
- **Предмет в мире (REQ-0014)** уточняет состояние `InWorld` из **REQ-0012** и служит домом для анимаций предмета в мире.
- **Мини-карта (REQ-0010)** и **Инвентарь (REQ-0011)** — раздельные HUD-виджеты в одной `HUD` (`main.tscn`).
- **Ядро (REQ-0003)** определяет мир/камеру/ввод, на которые опираются все остальные фичи.
