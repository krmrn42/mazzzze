## F-47: Теннисный мячик — свойства и многоразовость

Мячик специализировает базовый предмет ([REQ-0012](../REQ-0012-base-item/README.md)): конкретная модель, паттерн B и **не расходуется** при использовании. Главное отличие от фотоаппарата/фотографии (одноразовых) — бросок **не уничтожает** мяч.

### Тип предмета

| Параметр | Значение |
|----------|----------|
| Идентификатор типа | `tennis_ball` |
| Отображаемое имя | «Теннисный мячик» |
| Паттерн использования | **B** (Активация «в руку» с последующим использованием) |
| Расходуемость | **Многоразовый** — никогда не уничтожается при броске |
| 3D-модель в игровой зоне | `art/base_ball.glb` |

> Модель `base_ball.glb` визуально читается как бейсбольный мяч; дисплейное имя остаётся «Теннисный мячик» по роли из [IDEA-0025](../../ideas/items/IDEA-0025-tennis-ball.md).

### Бросок как переход Activated → InWorld

Бросок — это **новый вариант** перехода `Activated → InWorld` (рядом с `drop activated`, [F-15](../REQ-0012-base-item/01-states-logic.md)):

| Свойство | Значение |
|----------|----------|
| Что запускает | Заряженное использование активированного мячика (ЛКМ, см. [F-50](./04-input.md)) |
| Куда переходит мяч | В мир (`InWorld`) как физический снаряд, затем — как лежащий предмет ([F-48](./02-throw-logic.md)) |
| Расходование | **Нет** — мяч не уничтожается. Это ключевое отличие от `use activated` фотоаппарата/фотографии |
| Слот инвентаря | При броске слот **освобождается** (как при `drop activated`), бронь снимается |

### Цикл многоразовости

Мяч циркулирует по состояниям базового предмета без уничтожения:

```
InWorld (лежит/брошен) → [подбор, REQ-0016] → InInventory → [активация] → Activated
        ↑                                                                    │
        └──────────────────────── [бросок, F-48] ────────────────────────────┘
```

- После броска мяч оказывается в мире и **доступен для авто-подбора** (REQ-0016) при сближении игрока — как любой `WorldItem`.
- Подобранный мяч возвращается в свободный слот инвентаря; его можно снова активировать и бросить.
- Цикл повторяется неограниченно; мяч никогда не расходуется и не «теряется» от самого броска.

### Допустимые переходы (унаследованы + бросок)

| Переход | Направление | Примечание |
|---------|------------|-----------|
| Подбор (pickup) | InWorld → InInventory | Стандартный (REQ-0016) |
| Активация (activate) | InInventory → Activated | Стандартная, слот бронируется (F-19a) |
| Деактивация (deactivate) | Activated → InInventory | Стандартная; заряд сбрасывается (см. [F-50](./04-input.md)) |
| **Бросок (throw)** | **Activated → InWorld** | **Новый**: заряженный физический снаряд; мяч не расходуется |
| Выбрасывание (drop) | InInventory → InWorld | Стандартное (мягкая «звезда» `DropProjectile`, REQ-0015) — **не** бросок |
| Выброс активированного (drop activated) | Activated → InWorld | Стандартный (REQ-0015) — **не** бросок |

> Бросок и выброс — разные действия: бросок — это прицельный заряженный снаряд из руки; выброс — мягкая дуга-«звезда» из инвентаря/руки без заряда и без боевой функции.
