## F-39: Свойства шаблона монстра

Шаблон задаёт **набор** настраиваемых свойств; каждый конкретный тип монстра фиксирует свои значения. Свойства описаны ниже без чисел по умолчанию — умолчания живут в типе монстра (см. [REQ-0020](./REQ-0020-base-monster-wukong/03-params-data.md)).

### Идентичность

| Свойство | Назначение |
|----------|-----------|
| Идентификатор типа | Уникальный ключ типа монстра (связывает с моделью, поведением, режимом доставки) |
| Размер (вариант) | **Большой/обычный** (база) либо **маленький** (будущая специализация — убиваемый мячом). Влияет на то, как мяч воздействует на монстра ([IDEA-0025](../../ideas/items/IDEA-0025-tennis-ball.md)) |

### Восприятие (см. F-40)

| Свойство | Назначение |
|----------|-----------|
| Дальность зрения | Максимальное расстояние обнаружения |
| Угол зрения (FOV) | Полный угол конуса зрения |

### Движение

| Свойство | Назначение |
|----------|-----------|
| Скорость цикла | Скорость перемещения в нейтральном состоянии (патруль/блуждание) |
| Скорость преследования | Скорость перемещения в состоянии Угрозы (контактный режим) |

### Угроза (см. F-42)

| Свойство | Назначение |
|----------|-----------|
| Режим доставки | `Contact` или `Ranged` (выбирает тип) |
| Урон за попадание | Величина урона системе здоровья игрока за одно попадание |
| Порог прекращения преследования | Расстояние монстр↔игрок, при превышении которого (вместе с потерой видимости) преследование прекращается |
| Интервал контакта | Минимальное время между нанесениями урона при непрерывном касании (контактный режим) |

### Сбитие с толку (см. F-41)

| Свойство | Назначение |
|----------|-----------|
| Длительность стана | Сколько монстр стоит неподвижно после попадания мячом |
| Радиус отвлечения | На каком расстоянии предмет считается «достигнутым» в отвлечении |

### Визуальные (см. [05-visual.md](./05-visual.md))

| Свойство | Назначение |
|----------|-----------|
| Путь к 3D-модели | `.glb`-ассет конкретного типа |
| Целевой размер | Высота монстра в мировых единицах (масштабирование модели) |

## F-43: Реестр монстров

Все живые монстры в мире регистрируются в **едином реестре** (по аналогии с `WorldItem.All` для предметов).

**Назначение:**

- **Таргетинг для предметов:** брошенный мячик и иные предметы-диверсии ищут цель через реестр.
- **Нанесение урона монстру** (от будущих средств поражения) и применение состояний (стан/отвлечение).
- **Сканирование для игрока/мини-карты** (при будущей индикации угроз).

**Правила:**

- Монстр добавляется в реестр при появлении в мире и удаляется при уничтожении/исчезновении.
- Реестр содержит только живых монстров, находящихся в загруженной области мира (стриминг чанков не должен порождать «висящих» записей).
- Доступ к реестру — глобальный (статический), как у `WorldItem.All`.

**Связано:** состояния/таргетинг — [F-41](./01-states-logic.md); стриминг чанков — [REQ-0003-core/F-06](../REQ-0003-core/04-chunking-logic.md); аналог для предметов — `WorldItem.All` ([REQ-0014](../REQ-0012-base-item/REQ-0014-base-item-item-in-world/README.md)).
