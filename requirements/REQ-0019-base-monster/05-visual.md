# Базовый монстр — Визуальное представление (Visual)

> **Фича:** REQ-0019-base-monster · **Тип документа:** требования, грань «визуал» (WHAT, без деталей реализации)

## Визуальное представление монстра

Каждый тип монстра имеет 3D-модель и читается в масштабе коридора как подвижная угроза.

### 3D-модель в мире

| Свойство | Описание |
|----------|----------|
| Формат | Godot-совместимая 3D-модель (`.glb` — GLTF binary), своя для каждого типа |
| Размер | Масштабируется до целевой высоты типа. **Большой/обычный** размер (база) читается как крупная фигура, ощутимо больше предмета и сопоставимая с ростом игрока; **маленький** (будущая специализация) — заметно ниже |
| Размещение | Стоит на полу (низ модели у Y ≈ 0), перемещается по коридорам |
| Ориентация | Разворачивается по направлению движения/взгляда (смотрим туда, куда смотрит конус зрения — F-40) |
| Видимость | Различима при стандартном освещении сцены и с высоты камеры (вид сверху-сзади) |

### Коллизия

| Свойство | Описание |
|----------|----------|
| Тело монстра | Имеет физическую коллизию, достаточную для: **(а)** регистрации касания игрока (контактный урон, F-42); **(б)** попадания брошенного мяча/предмета-снаряда; **(в)** непроходимости для игрока (монстр — препятствие) |
| Размер тела | Соответствует габариту модели типа монстра |

### Анимации

На уровне шаблона не фиксируются. Конкретные типы могут использовать клипы из своих `.glb` (покой, ходьба, бег, стан) — см. тип монстра.

> Конкретная модель, размер и анимации — у конкретного типа: [REQ-0020-base-monster-wukong](./REQ-0020-base-monster-wukong/README.md).
