## F-42: Доставка угрозы и урон игроку

В состоянии **Угрозы** (F-41) монстр доставляет угрозу игроку по одному из двух режимов. Шаблон определяет оба режима; конкретный тип монстра выбирает один.

### Режимы доставки

| Режим | Описание |
|-------|----------|
| **Контактный (Contact)** | Урон наносится при **непосредственном касании** монстром игрока. Монстр перемещается к игроку, чтобы его достичь |
| **Дальний (Ranged)** | Воздействие на расстоянии (форма — снаряд / хитскан / аура-дрейн — определяется конкретным дальним типом). В базовом шаблоне режим заявлен, но его конкретика и реализация — за будущим дальним типом монстра |

### Урон системе здоровья

- Попадание угрозы (контакт или дальнее воздействие) наносит **урон системе здоровья игрока**.
- Размер урона за одно попадание — настроечный параметр (F-39); конкретное значение задаёт тип монстра.
- Сама система здоровья (запас, регенерация, смерть/респаун) — **отдельная будущая фича** и здесь не определяется; монстр лишь «сообщает» о попадании в игрока.

### Преследование (для контактного режима)

Контактный монстр, увидев игрока, **преследует** его:

- Двигается в сторону игрока со своей скоростью преследования.
- Преследование **прекращается**, когда выполняется условие потери:
  - игрок вышел из конуса зрения / пропал с прямой видимости (F-40), **и**
  - расстояние монстр↔игрок превысило **порог прекращения преследования** (настроечный параметр, F-39).
- После прекращения монстр возвращается в Cycle (F-41).

> Конкретное значение порога прекращения задаёт тип монстра. Для первого монстра (Ifrit) по умолчанию это **1 сегмент лабиринта** — см. [REQ-0020/F-46](./REQ-0020-base-monster-ifrit/02-behavior-logic.md).

### Частота контакта (контактный режим)

Чтобы одно касание не снимало урон каждый кадр, урон по контакту наносится **не чаще**, чем раз в заданный интервал (параметр шаблона, F-39). Между ударами монстр должен снова контактировать с игроком (или выдержана пауза).
