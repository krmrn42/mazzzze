## Код

| Файл | Зона ответственности |
|------|----------------------|
| `src/Minimap.cs` | HUD-виджет (`Control`), процедурная отрисовка через `_Draw`, ввод (Tab/Ctrl+wheel), обновление размера в `_Process` |
| `src/MinimapState.cs` | Память исследования: FIFO последних 1000 клеток, перманентный вход/выход, ref-count «открытых» клеток |

## Ключевые решения

- **Отрисовка без текстур** — всё через `DrawCircle`/`DrawRect`/`DrawArc`/`DrawColoredPolygon`. F-09 (полный пергаментный look) отложен.
- **Детекция входа в клетку — в `_PhysicsProcess`** (фикс. 60 Гц), НЕ в `_Process`: между тиками игрок проходит ≤ Speed·dt ≈ 0.08 у ≪ 3.6-метровой клетки, поэтому ни одна клетка не пропускается. `_Process` только меняет размер и зовёт `QueueRedraw`.
- **Туман войны** — FIFO на 1000 клеток (`MinimapState`), каждое посещение открывает окрестность 3×3 (`RevealRadius=1`); ref-count через `Dictionary<Vector2I,int>`, при вытеснении из хвоста вклад декрементируется. Вход/выход — отдельный перманентный набор.
- **Две зоны отрисовки:** ближняя (Чебышёв ≤ `NearRadius=7` ⇒ 15×15) с детализацией пол/стена через `MazeData.IsFloor`; дальняя — плоская силуэтная заливка.
- **Вращение:** `φ = -π/2 - atan2(fwd.y, fwd.x)`; по умолчанию `fwd` = planar camera-forward, при Tab — world-north `(0,-1)`.
- **Ввод:** Ctrl+wheel зумит (`_cellsRadius` ∈ [`MinCellsRadius 5`, `MaxCellsRadius 28`]) и поглощает событие; `Player` дополнительно игнорирует wheel при зажатом Ctrl, поэтому обычный wheel зумит камеру.
