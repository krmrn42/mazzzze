# Maze Explorer — Базовая сущность игрового предмета (Item)

> **Document type:** Feature requirements.
> **Audience:** Game designers, product owners, AI agents.
> **No implementation details** — this document describes WHAT the item entity is, not HOW.

---

## US-12: Базовая сущность игрового предмета

**Как** игрок,
**я хочу** чтобы в игре существовали предметы, которые можно находить в лабиринте, хранить в инвентаре, активировать (взять в руку) и использовать,
**чтобы** взаимодействовать с миром через собираемые ресурсы и инструменты.

**Критерии приёмки:**
- Предмет существует как самостоятельная игровая сущность
- Предмет может находиться ровно в одном из трёх состояний: в игровой зоне, в инвентаре, или активирован (в руке)
- Когда предмет находится в игровой зоне, он визуально отображается в 3D-мире
- Когда предмет находится в инвентаре, он отображается иконкой в соответствующей ячейке
- Когда предмет активирован, он отображается UI-индикатором у края экрана; слот инвентаря при этом **бронируется** (помечается как занятый/недоступный) — см. F-19a
- Предмет не может находиться одновременно в двух состояниях
- Предмет не существует «сам по себе» без привязки к миру, инвентарю или руке

## F-15: Сущность Item — состояния и переходы

Предмет всегда находится в одном из трёх состояний. Переходы между состояниями происходят через игровые механики.

### Состояния

| Состояние | Описание |
|-----------|----------|
| **В игровой зоне (InWorld)** | Предмет существует как 3D-узел в сцене лабиринта. Имеет мировую позицию (клетка + смещение). Не принадлежит ни одному игроку. |
| **В инвентаре (InInventory)** | Предмет принадлежит игроку. Занимает ровно один слот в рюкзаке (см. US-11). Не имеет представления в 3D-мире. |
| **Активирован (Activated)** | Предмет в руке игрока. Не занимает слот инвентаря. Отображается UI-индикатором у края экрана. Одновременно может быть активирован только один предмет. |

### Переходы

| Переход | Направление | Описание |
|---------|------------|----------|
| Подбор (pickup) | InWorld → InInventory | Игрок подбирает предмет из игровой зоны. Предмет удаляется из мира и появляется в свободном слоте инвентаря. Если свободных слотов нет — подбор невозможен. |
| Выбрасывание (drop) | InInventory → InWorld | Игрок выбрасывает предмет из инвентаря. Предмет удаляется из инвентаря и появляется в игровой зоне рядом с игроком. |
| Выбрасывание активированного (drop activated) | Activated → InWorld | Игрок выбрасывает предмет из руки. Предмет деактивируется и появляется в игровой зоне рядом с игроком. |
| Активация (activate) | InInventory → Activated | Игрок берёт предмет в руку. Предмет удаляется из ячейки инвентаря и переходит в состояние Activated. **Слот бронируется** — помечается как занятый/недоступный, в него нельзя подобрать другой предмет. Если в руке уже есть другой активированный предмет, он автоматически деактивируется (см. F-19). |
| Деактивация (deactivate) | Activated → InInventory | Игрок убирает предмет из руки обратно в инвентарь. Предмет возвращается в **забронированный слот**. Если предмет был уничтожен в состоянии Activated — слот освобождается (см. F-19a). |
| Использование из инвентаря (use immediate) | InInventory → InInventory или уничтожение | Предмет применяется напрямую из инвентаря (паттерн A, см. F-18). Последствия зависят от типа: расходник уничтожается, ключ/квестовый предмет остаётся. |
| Использование активированного (use activated) | Activated → Activated или уничтожение | Предмет применяется из руки (паттерн B, см. F-18). Последствия зависят от типа: оружие остаётся в руке, одноразовый предмет уничтожается. |

## F-16: Сущность Item — визуальная модель

Каждый тип предмета имеет визуальное представление в зависимости от текущего состояния.

### 3D-модель в игровой зоне (состояние InWorld)

| Свойство | Описание |
|----------|----------|
| Формат | Godot-совместимая 3D-модель (`.glb` — GLTF binary) |
| Размер | Умещается в пределах одной ячейки лабиринта (3.6 × 3.6) и ощутимо меньше персонажа; предмет — небольшая деталь в масштабе коридора |
| Размещение | На поверхности пола (Y ≈ 0), с небольшим смещением от центра клетки для избежания совпадения с позицией игрока |
| Коллизия | Предмет не блокирует движение игрока (нет физической коллизии как у стен). Допускается overlap-зона для обнаружения подбора |
| Видимость | Модель видна при стандартном освещении сцены; читается с высоты камеры (вид сверху-сзади) |

### Иконка в инвентаре (состояние InInventory)

| Свойство | Описание |
|----------|----------|
| Тип | 2D-изображение (PNG/SVG/Godot-текстура) |
| Размер | Соответствует размеру ячейки инвентаря (зависит от компоновки 3×4) |
| Стиль | Узнаваемый силуэт предмета; единообразный стиль всех иконок |
| Разрешение | Достаточное для чёткого отображения в сетке 3×4 (ячейка ~64×64 или ~96×96 пикселей) |

### UI-индикатор активированного предмета (состояние Activated)

| Свойство | Описание |
|----------|----------|
| Тип | Иконка предмета + компактная подпись у края экрана |
| Расположение | Рядом с инвентарной панелью (правый нижний угол) либо отдельный слот над ней |
| Видимость | Отображается только когда есть активированный предмет; исчезает при деактивации |
| Содержание | Иконка предмета и, опционально, отображаемое имя |

> **3D-модель в руке (состояние Activated):** на уровне базовой сущности предмета НЕ требуется. Конкретные типы предметов (оружие, инструменты) могут определить собственное 3D-представление в руке, включая положение на модели персонажа, ориентацию и анимации использования — это будет описано в требованиях соответствующих конкретных предметов.

Соответствие всех визуальных представлений — по идентификатору типа предмета: один тип = один набор (3D-модель + иконка инвентаря + иконка UI-индикатора).

## F-17: Сущность Item — базовая структура данных

Каждый экземпляр предмета (не тип, а конкретный предмет) имеет минимальный набор полей для идентификации и отображения.

| Поле | Тип | Назначение |
|------|-----|-----------|
| Идентификатор типа (ItemType) | Строковый или перечислимый ключ | Определяет, ЧТО это за предмет. Связывает предмет с его визуальными представлениями, поведением при использовании и паттерном использования (A или B, см. F-18) |
| Отображаемое имя | Строка | Название предмета (например, «Зелье здоровья», «Ключ от выхода») |
| Состояние | Перечисление: `InWorld`, `InInventory`, `Activated` | Текущее состояние предмета |
| Позиция в мире | Координаты клетки лабиринта + смещение | Где в лабиринте находится предмет. Заполнено **только** для `InWorld` |
| Индекс слота | Целое число 0–11 | Какой слот занимает в инвентаре. Заполнено **только** для `InInventory` |

**Особые условия:**
- `InWorld`: заполнена только «позиция в мире»; «индекс слота» пуст
- `InInventory`: заполнен только «индекс слота»; «позиция в мире» пуста
- `Activated`: предмет в руке; «позиция в мире» пуста; «индекс слота» заполнен — ссылается на забронированный слот (см. F-19a)
- Идентификатор типа неизменяем на протяжении жизни экземпляра
- Конкретные типы предметов и их поведение при использовании определяются в отдельных документах в `requirements/items/`

## F-18: Сущность Item — паттерны использования

Каждый тип предмета следует одному из двух паттернов использования. Паттерн определяется на уровне типа предмета (поле в данных типа, не экземпляра).

### Паттерн A: Немедленное использование (immediate use)

Предмет применяется напрямую из инвентаря, без предварительной активации. Активация для таких предметов недоступна (или бессмысленна).

| Свойство | Описание |
|----------|----------|
| Где применяется | Из инвентаря (состояние `InInventory`) |
| Ввод | Цифровые клавиши 1–4 в открытом инвентаре (см. US-11/F-14) |
| Результат | Эффект применяется немедленно. Расходник уничтожается, ключ/квестовый предмет остаётся |
| Примеры | Зелье здоровья, еда, свиток |

**Жизненный цикл паттерна A:**
1. Предмет в инвентаре
2. Игрок открывает инвентарь (двойное I)
3. Игрок выбирает ячейку цифровыми клавишами
4. Эффект применяется мгновенно
5. Если расходник — предмет уничтожается (слот освобождается)
6. Если не расходник — предмет остаётся в слоте

### Паттерн B: Активация с последующим использованием (activate then use)

Предмет сначала активируется (берётся в руку), затем используется отдельным действием.

| Свойство | Описание |
|----------|----------|
| Шаг 1: Активация | Предмет переводится из `InInventory` в `Activated`. Слот инвентаря освобождается |
| Шаг 2: Использование | Активированный предмет применяется отдельным вводом |
| Ввод активации | Цифровые клавиши 1–4 в открытом инвентаре (те же, что для паттерна A) — но вместо немедленного использования предмет активируется |
| Ввод использования | Левая кнопка мыши (ЛКМ) — применяет активированный предмет |
| Результат использования | Эффект применяется. Одноразовый предмет уничтожается (рука пустеет). Многоразовый предмет остаётся активированным |
| Примеры | Оружие (активировать → прицелиться → выстрелить), инструмент (активировать → применить) |

**Жизненный цикл паттерна B:**
1. Предмет в инвентаре
2. Игрок открывает инвентарь (двойное I)
3. Игрок выбирает ячейку цифровыми клавишами → предмет активируется (инвентарь можно закрыть или оставить открытым)
4. Игрок нажимает ЛКМ → эффект применяется
5. Если расходник (метательный предмет, одноразовый инструмент) — уничтожается
6. Если многоразовый (оружие) — остаётся активированным, можно применить снова
7. Деактивация возвращает предмет в инвентарь (см. F-19)

**Важное различие:** в открытом инвентаре одни и те же цифровые клавиши (1–4) работают по-разному в зависимости от типа предмета:
- Паттерн A: нажатие цифры = немедленное использование
- Паттерн B: нажатие цифры = активация (предмет переходит в руку)

## F-19: Правила активации

### Эксклюзивность

- Одновременно может быть активирован **только один** предмет
- При активации нового предмета, текущий активированный предмет автоматически деактивируется:
  - Предмет возвращается в **свой забронированный слот** (см. F-19a)
  - Если предмет был уничтожен в состоянии Activated — его слот уже освобождён; активация нового предмета бронирует слот нового предмета

### Деактивация

- Деактивация переводит предмет из `Activated` в `InInventory`
- Предмет возвращается в **свой забронированный слот** — свободный слот не требуется (он уже зарезервирован)
- Деактивация происходит:
  - Явно: игрок повторно выбирает тот же предмет в инвентаре (toggle: активация ↔ деактивация)
  - Неявно: при активации другого предмета
  - При выбрасывании: предмет из руки выбрасывается в мир (Activated → InWorld), слот освобождается

### Активация из мира

- Активировать предмет напрямую из мира (минуя инвентарь) **нельзя**
- Предмет должен быть сначала подобран в инвентарь (InWorld → InInventory), затем активирован (InInventory → Activated)

## F-19a: Бронирование слота активированного предмета

Когда предмет переходит в состояние `Activated`, слот инвентаря, который он занимал, **не освобождается** — он бронируется (помечается как занятый/недоступный).

### Правила бронирования

| Ситуация | Поведение |
|----------|-----------|
| Слот занят активированным предметом | Слот считается занятым. Подбор нового предмета в этот слот невозможен. Другие предметы не могут быть перемещены в этот слот |
| Уничтожение активированного предмета | Слот **освобождается**. Бронь снимается, слот становится доступным для подбора |
| Деактивация | Предмет возвращается в этот же слот. Бронь снимается, слот снова содержит предмет в обычном режиме |
| Выбрасывание активированного (Activated → InWorld) | Слот **освобождается**. Бронь снимается |
| Создание нового предмета при использовании | Если использование активированного предмета создаёт другой предмет, он должен занять этот же слот после уничтожения исходного |

### Индикация

- Забронированный слот в UI инвентаря визуально отличается от свободного: показывается специальным индикатором (например, значок «руки» или особая рамка), сигнализирующим что предмет находится в руке
- При открытии инвентаря игрок видит, какой слот зарезервирован за активированным предметом
