## US-17: Фотография — предмет-портал в запечатлённую точку лабиринта

**Как** игрок,
**я хочу** сделать фотографию старинным фотоаппаратом, а позже — с фотографией в руке войти в неё (пройти вперёд),
**чтобы** перенестись в ту точку лабиринта, где был сделан снимок, с тем же направлением взгляда.

**Критерии приёмки:**
- Фотография существует как самостоятельный игровой предмет, наследующий базовую сущность предмета (US-12)
- Фотография создаётся как результат успешного использования фотоаппарата (US-13) и помещается в освободившийся слот фотоаппарата
- Фотография неизменно хранит данные, полученные при создании: точку лабиринта, где был игрок, и направление взгляда (видоискателя) в момент съёмки ([F-32](./01-data-logic.md))
- Фотографию можно активировать (взять в руку) — паттерн B ([F-18](../REQ-0012-base-item/03-usage-logic.md))
- С активированной фотографией в руке игрок входит в неё, двигаясь вперёд непрерывно небольшое время при наличии достаточного пространства впереди ([F-33](./02-teleport-logic.md))
- По завершении «входа» игрок мгновенно перемещается в запечатлённую точку с запечатлённым направлением взгляда; фотография расходуется (уничтожается) ([F-33](./02-teleport-logic.md))
- Фотографию можно выбросить и подобрать как обычный предмет; при этом её запечатлённые данные сохраняются

## F-31: Фотография — свойства и жизненный цикл

Фотография наследует базовую сущность предмета (F-15..F-19) со следующими уточнениями.

### Тип предмета

| Параметр | Значение |
|----------|----------|
| Идентификатор типа (`ItemType`) | `photo` |
| Отображаемое имя | «Фотография» |
| Паттерн использования | **B** (Активация с последующим использованием) — вариант: «вход движением вперёд» (см. [F-33](./02-teleport-logic.md)) |
| Расходуемость | **Одноразовый** — уничтожается после перемещения |
| Создание | Результат успешного использования фотоаппарата ([REQ-0013, F-20](../REQ-0013-vintage-camera/README.md)) |
| 3D-модель в игровой зоне | фото-карточка (полароид); визуальный ассет — TBD (допустим плейсхолдер) |
| Иконка инвентаря | миниатюра запечатлённого вида (цель) или плейсхолдер (см. [F-34](./03-visual.md)) |
| UI-индикатор активированного | (определится при создании визуального ассета; допустим плейсхолдер) |

### Допустимые переходы

Все переходы — по правилам базовой сущности ([F-15, F-19](../REQ-0012-base-item/01-states-logic.md)):

| Переход | Направление | Условие |
|---------|------------|---------|
| Создание (срабатывание фотоаппарата) | → InInventory | Фотоаппарат в состоянии Activated завершил съёмку; фотография помещается в забронированный слот фотоаппарата |
| Выбрасывание (drop) | InInventory → InWorld | Стандартный. Запечатлённые данные сохраняются |
| Выбрасывание активированного (drop activated) | Activated → InWorld | Стандартный. Прерывает «вход» (см. [F-33](./02-teleport-logic.md)), данные сохраняются |
| Активация (activate) | InInventory → Activated | Стандартный, слот бронируется (F-19a) |
| Деактивация (deactivate) | Activated → InInventory | Прерывает «вход» (см. [F-33](./02-teleport-logic.md)) |
| Перемещение (use — walk-into) | Activated → (уничтожение) | Одноразовый: игрок вошёл в фотографию. Перенос в запечатлённую точку, фотография уничтожается, слот освобождается |

### Жизненный цикл

1. **Создание:** фотоаппарат завершает съёмку ([F-22](../REQ-0013-vintage-camera/02-timer-logic.md)) → создаётся экземпляр фотографии с запечатлёнными данными ([F-32](./01-data-logic.md)) → помещается в слот, освобождённый фотоаппаратом → фотоаппарат уничтожается
2. Фотография находится в инвентаре (InInventory). Может быть выброшена/подобрана (InWorld ↔ InInventory) — данные не меняются
3. Игрок активирует фотографию → она в руке (Activated), слот бронируется
4. Игрок входит в фотографию: двигается вперёд непрерывно в течение времени входа, имея впереди достаточно места ([F-33](./02-teleport-logic.md))
5. По завершении входа — перенос игрока в запечатлённую точку с запечатлённым направлением; фотография уничтожается, слот освобождается
