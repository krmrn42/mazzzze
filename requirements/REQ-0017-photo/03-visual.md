## F-34: Визуальное представление фотографии

Фотография имеет визуальные представления в духе старой фотографии/полароида, согласованные со стилистикой фотоаппарата ([REQ-0013](../REQ-0013-vintage-camera/README.md)).

### 3D-модель в игровой зоне (InWorld)

В дополнение к [F-16](../REQ-0012-base-item/04-visual.md):

| Свойство | Значение |
|----------|----------|
| Форма | Фото-карточка (полароид): прямоугольный картон с кадром и светлым полем; читается как «лежащая фотография» |
| Размер | Как базовый предмет: умещается в ячейке (3.6 × 3.6), ощутимо меньше персонажа |
| Размещение | На полу (Y ≈ 0), как выброшенный предмет ([REQ-0014](../REQ-0012-base-item/REQ-0014-base-item-item-in-world/README.md)) |
| Ориентация | Произвольная на полу |
| Ассет | TBD (допустим плейсхолдер) |

### Иконка в инвентаре (InInventory)

| Свойство | Значение |
|----------|----------|
| Назначение | Помочь игроку узнать, куда ведёт данная фотография |
| Идеал | **Миниатюра запечатлённого вида** — уменьшенное изображение того, что было в видоискателе при съёмке |
| Первый проход (плейсхолдер) | Единая иконка «фотография» для всех экземпляров (если миниатюра отложена) |
| Стиль | Под старую фотографию: сепия / приглушённый тон, виньетка по углам, лёгкая зернистость |
| Ассет | TBD |

### UI-индикатор активированной фотографии (Activated)

| Свойство | Значение |
|----------|----------|
| Назначение | Показать, что фотография в руке и вход активен (двигайся вперёд, чтобы войти) |
| Содержание | Иконка фотографии; опционально — подсказка «иди вперёд, чтобы войти» |
| Расположение/видимость | По правилам активированного предмета ([F-16](../REQ-0012-base-item/04-visual.md)) |

### Живое окно вида «сквозь фото» и индикатор входа

Пока фотография активирована, над головой игрока показывается **живое окно** вида запечатлённой точки — **не статичный снимок**: если в этот момент в том месте что-то движется (например, проходит монстр), это видно в окне. По мере накопления входа ([F-33](./02-teleport-logic.md)) окно **растёт** и смещается к центру экрана — «фотография увеличивается», подводя к переносу.

| Параметр | Значение |
|----------|----------|
| Что показывает | Живой вид запечатлённой точки + близость к срабатыванию (0 → `EnterDuration`) |
| Визуальные ориентиры | Окно над головой растёт от небольшого к почти полноэкранному по мере входа |
| При сбросе прогресса | Окно уменьшается обратно; при деактивации/выбросе — исчезает |

> Оформление окна единообразно с видоискателем фотоаппарата ([F-21](../REQ-0013-vintage-camera/01-viewfinder-ui.md)): та же рамка «под старину», сепия.

Точная реализация эффекта — на этапе проработки; первый проход может быть простым (например, тонкий индикатор-полоса + лёгкая виньетка).

### Переход перемещения (маскировка подгрузки чанков)

Так как точка перемещения может быть далеко от текущей позиции, после срабатывания нужен визуальный переход, скрывающий кратковременную подгрузку чанков (стриминг, [TECH_SPEC §5.2](../TECH_SPEC.md)):

| Параметр | Значение |
|----------|----------|
| Идея | «Вспышка/проявление» в духе старой фотографии: краткий уход в белый/чёрный (или сепия), смена сцены, обратное проявление уже в запечатлённой точке |
| Длительность | Короткая, достаточная, чтобы скрыть появление чанков |
| Стиль | Согласован с фото-темой (вспышка затвора / печать снимка) |
