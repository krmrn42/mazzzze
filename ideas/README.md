# Ideas — парковка идей (Maze Explorer)

Сырые идеи игровых механик, предметов, монстров, мира и т.п. Здесь живут **недетализированные** задумки. Когда идея созревает — она превращается в требование в [../requirements/](../requirements/README.md) (REQ-NNNN), а здесь помечается как promoted и ссылается на новый REQ.

## Соглашения

- **Папка = область:** `mechanics/`, `items/`, `monsters/`, `world/`, `progression/`, `atmosphere/` (можно заводить новые области). Внутри — плоско файлы идей. Одну папку на одну идею делать **не надо** (исключение — самодостаточная идея, которую хочется развернуть в подпапках).
- **Имя файла:** `IDEA-NNNN-<slug>.md`, где `NNNN` — 4-значный **глобально уникальный** номер. Следующий свободный = максимальный существующий + 1 (`ls ideas/*/IDEA-*`).
- **Объём:** допустима буквально одна строка. Идея фиксируется как искра, детали — позже.
- **Язык:** русский (как в требованиях).

## Шаблон файла

```markdown
# IDEA-NNNN — <Название>

> **Область:** <area> · **Статус:** 🟡 идея

<одна-две строки — в чём суть и почему интересно>

**Связи:** REQ-XXXX / IDEA-XXXX (если есть)
```

## Жизненный цикл

| Статус | Значение |
|--------|----------|
| 🟡 идея | сырая задумка, только что зафиксирована |
| 🔵 детализируется | идёт обсуждение/проработка (файл обрастает подробностями) |
| ✅ promoted | превращена в требование — заведён `REQ-NNNN`, здесь оставлена ссылка |
| ❌ отклонена | решили не делать; файл остаётся для истории (статус + причина) |

При **promote**: создать `requirements/REQ-NNNN-<slug>/`, в шапке IDEA-файла поставить ✅ и ссылку на REQ, в REQ design.md сослаться обратно на IDEA.
