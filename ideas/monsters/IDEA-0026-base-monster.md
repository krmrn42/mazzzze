# IDEA-0026 — Базовый тип монстра

> **Область:** monsters · **Статус:** 🔵 детализируется

Фундаментальный **шаблон** монстра, от которого специализируются конкретные типы (размер, доставка, скорость). Базовый размер — **большой/обычный**: мячик ([IDEA-0025](../items/IDEA-0025-tennis-ball.md)) его **путает, но не убивает**. Маленький (убиваемый мячом) — будущая специализация.

## Основа шаблона

| Ось | Значение |
|-----|----------|
| Размер | **большой/обычный** (база); «маленький» — будущая специализация (мячом можно убить) |
| Восприятие | **Конус зрения + проверка прямой видимости стенами** (FOV + дальность + LoS). Определяет «видит игрока» / «видит мячик» |
| Скорость | **настраивается** (параметр), своя для каждого конкретного типа |
| Доставка угрозы | **режим в шаблоне**, две ветви: **контактный** (угроза при касании игрока) и **дальний** (воздействие издалека). Конкретный тип выбирает режим |
| Урон игроку | наносит **урон системе здоровья** игрока (и контактный, и дальний). Сама система здоровья — отдельная будущая фича (см. TODO) |

## Поведение (каркас)

- **Цикл по умолчанию** (игрока не видит): базовое перемещение (патруль/блуждание — конкретика за конкретным типом).
- **Заметил игрока** в конусе зрения с прямой видимостью → переходит к угрозе (по своему режиму доставки: контактный — идёт/бежит к игроку; дальний — действует на расстоянии).
- **Получил попадание мячом** → сбит с толку: **стан** на время (замирает) и/или **отвлечение** на мячик, если монстр не видел броска (см. [IDEA-0025](../items/IDEA-0025-tennis-ball.md)).
- После стана/отвлечения — возвращается к циклу.

## Состояния (набросок)

```
Цикл (поиск/патруль)  →  [увидел игрока]  →  Угроза (доставка)
        ↑                                              │
        └── [стан/отвлечение от мяча] ← [попал мячом] ──┘
```

## Восприятие — детали для мяча

- «**Не видел момент броска**» = игрока **не было в конусе зрения** монстра в момент броска.
- «**Видит мячик**» = мячик попал в конус зрения и есть прямая видимость.
- Эти две проверки рулят, какой из эффектов мяча сработает (стан при прямом попадании vs отвлечение на упавший мяч).
